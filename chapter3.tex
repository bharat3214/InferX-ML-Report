\chapter{Project Design}
\vspace{-1cm}
\par Project Design serves as the foundational blueprint for developing a software solution, outlining the overall structure, essential features, and technical specifications that guide the entire development process. The architectural decisions, data flow patterns, and component interactions are carefully planned to ensure scalability, maintainability, and alignment with the project's objectives. The Features and Functionality part details the specific capabilities and user-facing components of the project, explaining what the system will do and how users will interact with it. This comprehensive breakdown of core modules ensures the final product is a centralized and efficient platform that aligns with the project's goals and user needs. 

\section{Features and Functionality}
\begin{itemize}
\item \textbf{Automatic Data Analysis \& Preprocessing:}

The platform automatically analyzes the structure of uploaded datasets and performs necessary preprocessing steps to prepare the data for training. This includes handling missing values, formatting features, and ensuring the dataset is clean and suitable for machine learning workflows. By automating this process, it reduces manual effort and allows users to focus more on model building rather than data preparation.\\

\item \textbf{Multi-Model Training:}

The system supports training multiple machine learning models simultaneously based on the given dataset. This enables users to explore different algorithms without manually configuring each one, ensuring a comprehensive approach to model development and improving the chances of achieving optimal performance.\\

\item \textbf{Best Model Selection:}

After training multiple models, the platform automatically evaluates their performance and selects the most accurate one. This eliminates the need for manual comparison and ensures that the best-performing model is chosen for further use.\\

\item \textbf{Model Export (.pkl):}

Once the best model is selected, the system generates a reusable \texttt{.pkl} file. This file can be easily integrated into applications or deployed in real-world systems, making the transition from development to production seamless.\\

\item \textbf{Prediction Interface:}

The platform provides a simple and user-friendly interface where users can input data and receive predictions from the trained model. This makes machine learning accessible even to users with minimal technical expertise.\\

\item \textbf{Dataset Upload \& Processing:}

Users can upload their datasets directly to the platform, where the system processes and prepares the data for model training. This feature ensures a smooth and efficient workflow from data ingestion to model development.\\

\item \textbf{Automatic Model Training:}

Based on the type and structure of the dataset, the platform automatically initiates training using suitable machine learning models. This reduces the need for manual configuration and speeds up the development process.\\

\item \textbf{Model Evaluation \& Selection:}

The system evaluates all trained models using relevant performance metrics and compares their results. It then selects the best-performing model, ensuring accuracy and reliability in predictions.\\

\item \textbf{Model Deployment (.pkl):}

The selected model is converted into a deployable \texttt{.pkl} file, allowing users to integrate it into real-world applications, APIs, or systems with ease.\\

\item \textbf{Prediction System:}

The platform enables users to input new data and obtain predictions through an intuitive interface. This feature ensures that the trained model can be effectively used for practical decision-making and real-time applications.\\


\end{itemize}

\section{System Architecture}

    \centering

    \fbox{\includegraphics[width=16cm, height=10cm]{images/Designer.png}}
    \caption{System Architecture of InferX-ML}
    \label{fig:image1}
    \vspace{0.2cm}
    \parbox{\textwidth}{\small The diagram illustrates the architecture of InferX-ML, an automated machine learning pipeline that streamlines the entire lifecycle of model training and inference. At the top layer, users interact with the React-based frontend dashboard, which provides an intuitive interface for dataset uploads, experiment configuration, and prediction requests. The frontend communicates with the Flask REST API backend, which serves as the central orchestrator responsible for routing requests, managing user sessions, and coordinating asynchronous tasks via Celery workers backed by Redis as the message broker. The core ML Engine, built entirely in Python, handles data preprocessing, feature engineering, multi-model training, hyperparameter tuning, and model evaluation. Trained models and preprocessing pipelines are serialized as pickle artifacts and stored in MinIO object storage, while metadata such as experiment history, user records, and job statuses are persisted in PostgreSQL. The inference layer loads the best-performing model on demand to serve real-time predictions through a dynamically generated Streamlit interface, where users can input feature values and receive predictions accompanied by SHAP-based explainability visualizations. Additionally, the AI Insight Layer integrates with external APIs such as Google Gemini and Groq to enable natural language goal analysis, allowing users to describe their modeling objectives in plain English. This layered, microservices-oriented architecture ensures clear separation of concerns, horizontal scalability, and seamless deployment through Docker and Docker Compose.}
\end{figure}

    \parbox{\textwidth}{\small This Data Flow Diagram (DFD) at Level 2 provides a granular view of the modular, microservices-based architecture of the InferX-ML platform. At this level, each major subsystem identified in Level 1 is decomposed into its constituent services and their internal data flows. The Auth Service manages user registration, login, and session token validation, communicating directly with PostgreSQL to store and retrieve credential data. The Dataset Management Service handles file uploads, format validation for CSV and Excel files, and metadata extraction, persisting raw datasets in MinIO object storage while recording dataset metadata in PostgreSQL. The Training Orchestration Service coordinates the end-to-end model training workflow by dispatching asynchronous jobs through Celery workers, managing job queues via Redis, and tracking experiment progress and status updates in the database. The ML Engine Service encapsulates the core machine learning logic, including automated preprocessing, feature engineering, multi-algorithm training, cross-validation, hyperparameter tuning, and model evaluation using metrics such as accuracy, F1-score, RMSE, and silhouette score. Once the best-performing model is identified, it is serialized along with its preprocessing pipeline into a pickle artifact and stored in MinIO. The Prediction Service loads the trained model on demand and serves real-time predictions through dynamically generated input forms, accompanied by SHAP-based explainability reports. Finally, the AI Insight Layer communicates with external APIs such as the Gemini API to interpret user goals expressed in natural language and translate them into actionable pipeline configurations, bridging the gap between domain expertise and technical ML workflows.
}
\end{figure}

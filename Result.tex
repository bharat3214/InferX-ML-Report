\chapter{Results }\vspace{-1cm}

The successful realization of the InferX-ML platform is demonstrated through a comprehensive visual analysis of the integrated system. The following sections offer a detailed walkthrough of the final user interface, highlighting the seamless convergence of automated machine learning workflows with modern web architecture. These screenshots serve as empirical evidence of the platform's robust functionality, ranging from intuitive data ingestion and goal analysis to high-performance model deployment and real-time visualization.\vspace{0.4cm}

The home page is the first thing users see when they visit the platform. It clearly communicates the main idea: building ML models without writing any code. The layout is clean with a dark theme, and the two main buttons guide users to either get started right away or learn more about how the platform works.

This section gives users a clear picture of what the platform can do. Features like Smart Data Profiling, Explainable AI, and One-Click Export are all listed out. Each one handles a specific part of the ML process automatically, so users do not have to deal with things like encoding or outlier removal on their own.

The login page lets users sign in to their personal workspace where all their experiments and datasets are saved. It also reminds users of the platform's core promise, which is building powerful ML models without needing to write any code. The design is simple and straightforward.

After logging in, users land on the dashboard where they can see a summary of their work at a glance. It shows how many datasets they have, how many models have been trained, and if any training jobs are currently running. The quick action cards make it easy to jump straight into uploading data, starting a new training job, or making predictions.

The Datasets page shows all the files a user has uploaded. In this case, the Crop Recommendation dataset is visible with details like the number of rows, columns, and total file size. Some datasets can also be fetched directly from Kaggle, which is useful for users who want real-world data without having to search for it manually.

The training setup page is where users start a new experiment. They give it a name, pick a dataset, and optionally describe their goal in plain language. The platform then uses that description to suggest which column to predict and what type of model to use, saving a lot of manual configuration.

Once training starts, this page gives users a live view of what is happening. A progress bar shows which stage the pipeline is in, and the developer log on the right side prints out real-time updates as each algorithm is tested. This makes the whole process transparent rather than just showing a loading spinner.

When the training finishes, the platform shows a summary of what it found. In this case, Random Forest came out on top with an accuracy of 99.32 percent. The result is shown clearly so users know right away which model performed best and how well it did, without needing to dig through any logs.
